📋 待讨论事项 (6 项)

Q1 需要决策 各平台反爬策略与频率控制粒度

CSQAQ / SteamDT / DMarket 三个平台的反爬策略不同，频率限制也不同。

先保守后激进
默认 5min 间隔，逐步缩短。爬虫加随机延迟和 UA 轮换。被 ban 后自动降频。
调研先行
每个平台先手动测试，确定最佳频率后写死。初期不做自适应。
Q2 需要决策 饰品名称跨平台标准化

同一饰品在不同平台名称可能不同。

别名映射表
维护 CSV/JSON 文件，手动建立跨平台名称映射。每发现新别名追加。
模糊匹配
用相似度算法自动匹配，降低维护成本但可能出错。
Q3 功能选择 是否需要日/周汇总报告？
V1 跳过，V2 再加
初期聚焦实时告警。汇总报告不阻塞核心功能。
V1 包含简单日报
每日一次推送持仓市值和涨跌。实现成本低。
Q4 策略选择 价格数据保留多久后清理？
保留 30 天原始 + 聚合历史
30天内保留每条快照，超过30天只保留每小时均价。
全量保留
SQLite 单表百万行内性能尚可，一年约 10 万条/饰品。
Q5 后续功能 是否需要 Web 看板？
V1 纯企微交互，V2 Web 看板
初期通过企微机器人指令交互。
V1 包含极简 Web
简单 FastAPI 页面显示当前价格和告警。
Q6 用户界面 用户如何配置监控、查看持仓？
企微机器人指令
通过企微群内指令交互（/watch /list /alert）。用户已在企微。
Web 管理页面
独立 Web 页面配置管理。体验更好但需前端开发。
配置文件 + 企微通知
编辑 YAML 配置。初期最简单但用户体验差。
